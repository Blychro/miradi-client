Develop Project Budgets: Accounting Codes & Funding Sources

To develop a budget, you will first want to create accounting codes and funding sources.

On the <Accounting Codes> tab, if your project already has accounting codes from your home organization, you can use the <Import Accounting Code> button to import a tab delimited text file (2 columns, code followed by description). If you have to create codes from scratch, click on the <Create Accounting Code> button to add a new one and then type the code and the label in the box at the bottom of the page. Repeat for other codes as needed.

On the <Funding Sources> tab, click on the <Create Funding Source> button to add a new one and then type the code (if needed) and the label in the box at the bottom of the page. Repeat for other sources as needed.



Accounting codes are a set of categories developed by an organization to categorize and track expenses. They typically include hierarchical strings of numbers and/or letters (the code) along with a text description. For example:

200.000 Core Salaries

201.000 Director's salaries

202.000 Staff salaries

Information that is tracked through the use of accounting codes can include:

  • Expense Type - Salary, travel, equipment, etc. This information is often used to help prepare reports to donors and/or tax returns.

  • Department - The branch of the organization that is incurring the expense.

  • Funding Source - Some organizations also use their codes to specify the specific funding source (although Miradi also treats this separately).

  • Other Information - Categories that the organization wants to track.

It is likely that your home organization will already have a chart of accounts that they will want you to use.

Funding sources are, as the name implies, the sources of money available to your project. They are typically divided into:

  • Restricted sources - Funds such as a grant from a donor that are dedicated to a specific project or set of activities

  • Unrestricted sources - Funds that can be used in any way the project chooses
